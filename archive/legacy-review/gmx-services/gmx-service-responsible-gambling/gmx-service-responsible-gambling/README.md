# GMX Responsible Gambling Service

## Akka and Flink based Cloudflow Application

### Problem Definition

We ingest events generated by customer activity (e.g. `CustomerLoggedIn`, `DepositLimitSet`, `SportsBetPlaced`) and use those events to identify potential problem-gambling behviour by generating 'Alerts'.

These generated Alerts are sent to TruNarrative where the business rules for identifying customers with problematic behaviour (and interacting with them regarding their gambling behaviour) are handled.

### Sub projects

The following sub-projects constitute the whole application:

* `datamodel` - contains the Avro schema for all events
* `generators` - the Flink streamlets which generate the various alerts from customer event input
* `handlers` - the Akka streamlet used to send generated alerts to TruNarrative
* `calculators` - the Flink streamlets which provide pre-processing/aggregation of customer events (for instance calculating average stakes for bets)
* `pipeline` - the entry point containing the blueprint definition and local config

### Build the application

Here's the sequence of steps that you need to follow:

```
$ sbt
$ clean
$ buildApp
```

The above command builds the application and publish application Docker images to the Docker registry, as configured in `target-env.sbt`.

> **Note:** You need to copy `target-env.sbt.example` to `target-env.sbt` with appropriate settings for the Docker registry in order for the build and publish to go through.

For instance, to build to dockerhub you'd use:
```
ThisBuild / cloudflowDockerRegistry := Some("eu.gcr.io")
ThisBuild / cloudflowDockerRepository := Some("my-awesome-project")
```

The `buildApp` command, if successful, will publish the exact command to use for deployment in the cluster.

### Running Locally

To run the system locally you'll need to have Kafka available and uncomment the block at the bottom of the `pipeline/src/main/blueprint/blueprint.conf` file to ensure Cloudflow can find the unmanaged topics. You can then publish data to Kafka (e.g. with `kafkacat` or by running the gmx-service-customer service locally and interacting with that)
